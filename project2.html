STUDENT DATABASE MANAGEMENT USING OOPs

Desciption

Overview :-


The Student Database Management System is a sophisticated application developed
using Object-Oriented Programming (OOP) principles in Python, with data stored in
JSON files. This project highlights my ability to design and implement a system
that efficiently manages student records for both 10th and 12th grade students,
allowing for CRUD (Create, Read, Update, Delete) operations on student data.

Features :-



Add Student: Add new student records, including details such as ID, name, age, grade, and subjects.

Update Student: Update existing student information to reflect changes in academic or personal details.

View Student: Retrieve and display the details of any student by their unique ID.

Delete Student: Securely remove student records from the system.

Data Persistence: Use JSON files to store and manage student data, ensuring persistence across sessions.

Technologies Used :-



Programming Language: Python

Concepts: Object-Oriented Programming (OOP), JSON for data storage

Libraries: json, os for file handling

Key Components :-


Student Class: Defines the student entity with attributes such as ID, name, age, grade, and subjects. Implements a __str__ method for easy representation of student details.

StudentDatabase Class: Manages the collection of student records, providing methods for adding, updating, viewing, and deleting students. Handles data persistence using JSON files.

Project Highlights :-



Efficient Data Management: Utilizes dictionaries for quick access to student records by their unique IDs.

Data Persistence with JSON: Ensures that student data is saved to and loaded from JSON files, maintaining data integrity across sessions.

Modular Design: Separation of Student and StudentDatabase classes promotes code reusability and maintainability.

Scalability: Designed to handle a growing number of student records, making it suitable for large educational institutions.
